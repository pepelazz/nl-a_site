Создание и управление тестами на сервисе NL!A EduQuiz

1. Простой способ создания тестов

Тесты создаются на сервисе NL!A EduQuiz автоматически при загрузке вопросов в формате электронной таблицы


Детальная инструкция по данном способу создания тестов доступна по данной ссылке


2. Ручной способ создания тестов через админку сервиса NL!A EduQuiz

Для создания теста непосредственно на сервисе EduQuiz необходимо авторизовать (через профиль в одной из социальных сетей)


Затем нажать на кнопку “Create” (создать)



Затем в меню (слева вверху) нажать на вкладку “Редактировать задания”





В разделе редактирования задания для создания нового теста - нажать кнопку “Плюс” (справа внизу)

В открывшемся окне создания нового теста заполнить необходимые поля, выставить статус, а также указать необходимые параметры тестирования

Название теста - название задания, каким будут видеть его ученики

Статус - текущее состояние теста в контексте доступности ученикам для прохождения:

  1. Черновик - тест недопуступен для прохождения ученикам; возможно последующее “открытие” теста на занятии для его прохождения

  2. Открыто для выполнения - тест доступен для прохождения ученикам

  3. Закрыто для выполнения - тест закончен; прохождение недоступно

Тэги вопросов

За счет указания тэгов происходит “наполнение” теста вопросами

Каждый вопрос в сервисе NL!A EduQuiz также содержит атрибут “тэг”

За счет указанных при создании теста тэгов произойдет наполнение теста вопросами

ПРИМЕР: При создании теста с тэгом "математика" в тест войдут ВСЕ вопросы, загруженные в сервис NL!A EduQuiz и содержащие (размеченные) тэг "математика"

ПРИМЕР: При создании теста с тэгом “математика” и “сложение” в тест войдут ВСЕ вопросы, загруженные в сервис NL!A EduQuiz и содержащие (размеченные) тэг “математика” и “сложение”. Вопросы без тэга “сложение” в тест включены не будут

В указанном примере теста по охране труда вопросы с тэгами “охрана_труда” и “уровень_1” задаваться на будут, поскольку тэгом теста является такжэ тэг “начало” (отсутствует в нижнем блоке вопросов на иллюстрации)

Детально логика связи между “тестом” и “вопросами” за счет механики тэгов раскрыта в разделе по редактированию вопросов (п.6. Тэг (тема) вопрос)

Настройка параметров проведения теста:

Опция “Вопросы в случайном порядке”

Показывать вопросы в случайном порядке. Если данный пункт выбран, то с высокой долей вероятности каждому ученику в один момент времени будут показаны разные вопросы

ЛАЙФ-ХАК: Использование режима “показывать всем один вопрос” в совокупности с разрешением ученикам обсуждать вслух задаваемые вопросы на занятии - приводит к значительной прибавке позитива и энергии на занятии во время тестирования

Опция “Не показывать правильные ответы”

(Не)Показывать правильные ответы: задает параметр - выводить или нет правильный ответ ученику в конце каждого его ответа на вопрос

Сценарий с условием “не показывать ответ” стимулирует учеников к самостоятельному изучению темы теста (в случае большого количества ошибок)

Сценарий с условием “показывать ответ” способствует к дополнительному приобретению знаний ученика во время тестирования (ошибся, но узнал правильный ответ; вероятность закрепления знания - увеличивается)

Опция “Не показывать результат ответа”

Данная опция задает условие - выдавать или нет обратную связь ученику после каждого его ответа. При этом данное условие определяет обратную связь в виде отражения статуса “Верно” (палец вверх; + балл) или “Неверно”
